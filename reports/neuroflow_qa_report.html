Executive summary

The pipeline processed 100 synthetic subjects. It marked 62 scans as pass and 38 scans as flag. The goal is not diagnosis. The goal is engineering reliability: can the system read files, check quality, explain failures, and create traceable outputs?

Medical safety note: This project uses synthetic data only. It is not a diagnostic medical device and should not be used for clinical decisions.

Explain it like I am new

T1 MRI is like a clear still photo of the brain structure. fMRI is more like a short movie of signal changes over time. A QC pipeline checks whether the photo is clear, whether the movie is stable, and whether the person moved too much during recording.

NIfTI is the file format we use for these brain images. Think of it as a special box that stores both the image pixels and important image information. BIDS-style organization is like arranging school folders consistently: subject folder, anatomy folder, function folder, metadata file. That way another engineer can understand the project quickly.
